1st Thing: Git != GitHub

Git is a revision control system, a tool to manage your source code history. It is strictly a command-line tool.

GitHub is a website where you can upload a copy of your Git repository. It offers all of the functionality of Git as well as adding its own features.

Basic Git Overview

The purpose of Git is to manage a project, or a set of files, as they change over time. Git stores this information in a data structure called a repository.

A Git repository contains, among other things, the following:

  • Objects (code, text, whatever) you want to track
  • References to these objects, called heads

The Git repository is a hidden sub-folder in your project folder, called .git. You probably won't have to touch this ever.

Basic Git Overview: Branches

Git works on branches, which represent independent lines of development. By default, everyone's repositories are on a "master" branch.

Setting Up Git

Let's get on the command line and start!

  • On Windows, in the start menu search, type Git Bash
  • On Mac OSX, search terminal from spotlight search

Then type the following as separate commands:


git config --global user.name "Vicky Steeves"
git config --global user.email "vicky.steeves@nyu.edu"
git config --global color.ui "auto"
git config --global core.editor "gedit"
git config --list # lists your configurations

Where it says core.editor, put in your favourite plain text editor. This could be simply Notepad (Windows) or TextWrangler (Mac), but NOT Word.

Avoid making a mess and losing track of things! https://github.com/JonasAtWork/jonasatwork.github.io

Create a Folder & Repository

To create a blank repository, we have to create a new project folder and then initialize Git inside of it.

Type the following as separate commands in your command line:


cd Desktop 
mkdir hello-world 
cd hello-world 
git init

You've just created an empty Git repository!

Viewing Revision History

To review what you've been up to, type this in the terminal:


git log

This will list your commits with their IDs, date/time of commit, and commit messages. If you want to only look at the changes to a specific file, enter this command in the terminal:


git diff hi.txt

This will do the same as above, at a file-specific level.

Avoid making a mess and losing track of things! https://github.com/JonasAtWork/jonasatwork.github.io

Viewing Different Commits

Remember that weird number from git log with the commits? The first 7 characters of this ID number is referred to as the HEAD. You can view historys of objects this way too. Type in the terminal:


git diff 7a9209f # first 7 characters

This brings a history of everything since that version.

Avoid making a mess and losing track of things! https://github.com/JonasAtWork/jonasatwork.github.io

Rewriting History

The same 7 characters from before can be used to revert to an earlier version, using the command below:


git checkout HEAD objectname

checkout is used to tell Git to revert the object back the version listed.
